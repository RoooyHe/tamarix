# 补全测试体系

## 方案

使用 **Vitest** 作为测试框架（与 Vite 深度集成，零配置），按模块依赖复杂度分三层实施：

### 第一层：纯函数单元测试（无需 mock）

| 测试文件 | 被测模块 | 测试内容 |
|----------|----------|----------|
| `src/lib/matrix/workflow.test.ts` | `workflow.ts` | canTransition 所有状态转换组合、getAllowedNextStatuses、边界情况 |
| `src/lib/matrix/errors.test.ts` | `errors.ts` | getMatrixErrorCode/message/data/formatMatrixError 各种输入形态 |
| `src/lib/utils/url.test.ts` | `url.ts` | isValidUrl（合法/非法 URL）、sanitizeUrl（协议补全） |
| `src/lib/i18n/index.test.ts` | `i18n/index.ts` | t() 翻译、插值、fallback、setLocale、registerLocale |
| `src/lib/reports.test.ts` | `reports.ts` | getBurndownData、getStatusDistribution、getTrendData、getAssigneeWorkload、getVersionProgress |

### 第二层：带 mock 的单元测试

| 测试文件 | 被测模块 | Mock 对象 |
|----------|----------|-----------|
| `src/lib/stores/task-cache.test.ts` | `task-cache.ts` | mock MatrixClient、Room（含 currentState.getStateEvents） |
| `src/lib/stores/task-writes.test.ts` | `task-writes.ts` | mock MatrixClient + task-cache（验证乐观更新、回滚） |
| `src/lib/matrix/ticket-id.test.ts` | `ticket-id.ts` | mock MatrixClient.getRooms() |

### 第三层：基础设施

| 文件 | 操作 |
|------|------|
| `package.json` | 添加 `vitest` devDependency，添加 `"test": "vitest run"` 和 `"test:watch": "vitest"` 脚本 |
| `vite.config.ts` | 添加 `test` 配置（include、environment） |
| `.gitignore` | 添加 `coverage/` |
| `eslint.config.js` | 添加 test 文件的 eslint 配置 |

## 步骤

### 1. 安装依赖

```sh
bun add -d vitest
```

### 2. 配置 vitest

在 `vite.config.ts` 添加 test 配置：

```ts
import { sveltekit } from '@sveltejs/kit/vite';
import tailwindcss from '@tailwindcss/vite';
import { defineConfig } from 'vite';

export default defineConfig({
	plugins: [tailwindcss(), sveltekit()],
	test: {
		include: ['src/**/*.{test,spec}.ts']
	}
});
```

### 3. 更新 package.json scripts

```json
"test": "vitest run",
"test:watch": "vitest"
```

### 4. 更新 eslint.config.js

为 test 文件添加 eslint 环境配置（允许 describe/it/expect 全局变量）。

### 5. 编写测试文件

#### `src/lib/matrix/workflow.test.ts`
- 测试 canTransition：所有合法转换返回 true
- 测试 canTransition：所有非法转换返回 false
- 测试 canTransition：相同状态转换（no-op）返回 true
- 测试 getAllowedNextStatuses：每个状态返回正确的允许目标

#### `src/lib/matrix/errors.test.ts`
- 测试 getMatrixErrorCode：errcode 在顶层 / data 内 / 不存在
- 测试 getMatrixErrorMessage：error 在顶层 / data 内 / message fallback
- 测试 getMatrixErrorData：有 data / 无 data / 空对象
- 测试 formatMatrixError：有 code / 无 code

#### `src/lib/utils/url.test.ts`
- 测试 isValidUrl：http/https 合法、ftp/无协议/空字符串非法
- 测试 sanitizeUrl：//开头补 https:、无协议域名补 https://、已有协议不变

#### `src/lib/i18n/index.test.ts`
- 测试 t()：返回正确翻译、key 不存在返回 key 本身
- 测试 t()：{{param}} 插值替换
- 测试 setLocale / getCurrentLocale：切换语言
- 测试 registerLocale：注册自定义翻译

#### `src/lib/reports.test.ts`
- 构造 mock Task[] 数据
- 测试 getStatusDistribution：正确计数各状态
- 测试 getAssigneeWorkload：正确分组并按数量排序
- 测试 getTrendData：返回正确天数的数据点
- 测试 getBurndownData：空任务返回空数组
- 测试 getVersionProgress：按 version 分组统计

#### `src/lib/stores/task-cache.test.ts`
- 构造 mock MatrixClient 和 mock Room
- 测试 createTaskCache：upsert/getTasks/getByRoomId/getByProject/getById
- 测试 remove：删除后各查询返回空
- 测试 projectTaskIds 索引：任务切换 project 时旧索引被清理

#### `src/lib/stores/task-writes.test.ts`
- 构造 mock cache + mock client
- 测试 applyLocalPatch：patch null 回滚、patch 正常应用
- 测试 updateTaskStatus：合法转换调用 patchTask、非法转换返回 error

#### `src/lib/matrix/ticket-id.test.ts`
- 构造 mock client.getRooms() 返回含 ticket_id 的 room
- 测试 generateNextTicketId：从 max+1 生成
- 测试空项目：返回 TAM-1

### 6. 更新 .gitignore

添加 `coverage/`。

## 验证

```sh
bun run test          # 所有测试通过
bun run check         # 类型检查通过
bun run build         # 构建成功
```
