## 推荐下一步

### 1. CI/CD（最高优先级）

创建 `.github/workflows/ci.yml`，PR/push 触发：

```yaml
name: CI
on: [push, pull_request]
jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - run: bun install
      - run: bun run check      # typecheck
      - run: bun run lint        # eslint
      - run: bun run format:check # prettier
      - run: bun run test        # vitest
      - run: bun run build       # production build
```

### 2. Git Hooks

```sh
bun add -d husky lint-staged
npx husky init
# .husky/pre-commit: bun run lint && bun run format
# package.json lint-staged: { "*.{ts,svelte}": ["eslint --fix", "prettier --write"] }
```

### 3. CODE_OF_CONDUCT.md

使用 Contributor Covenant 标准模板。

### 4. 错误边界

创建 `src/routes/+error.svelte` 全局错误回退页。

### 5. 根目录 .env.example

列出前端所需环境变量。
