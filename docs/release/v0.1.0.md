# Tamarix v0.1.0

## Overview

Tamarix v0.1.0 is the first official release of a Matrix-based task management system, delivering core features across MVP (P0), Kanban view (P1), and intelligent front-end fallback layer (P2).

Core concept: **One task = One Matrix Room**, providing built-in real-time collaboration, federated communication, and end-to-end encryption.

---

## Features

### P0 — Minimum Viable Product (MVP)

- **Matrix Authentication** — Login with Homeserver URL / username / password, token persistence, auto-reconnect
- **Project Management** — Projects mapped from Matrix Spaces, create/view/switch projects
- **Task CRUD** — Tasks mapped from Matrix Rooms, create/view/edit tasks
- **Task Attributes** — Status (todo/in_progress/review/done/closed), Priority (critical/high/medium/low), Type (bug/feature/task/improvement/epic), Due date, Estimate/Story points, Tags
- **Task Detail Page** — Metadata editing, inline comment chat (based on Room Timeline)
- **Real-time Sync** — Listen on Matrix ClientEvent.Sync, auto-refresh on PREPARED/SYNCING

### P1 — Kanban View

- **Kanban Drag & Drop** — KanbanBoard / KanbanColumn / KanbanCard, drag to change task status
- **Command Palette** — Quick search tasks, switch projects, execute actions (Ctrl+K)
- **Project Task List** — Filtering and sorting support
- **Dashboard** — My tasks, upcoming deadlines, statistics

### P2 — Intelligence (Front-end Fallback Layer)

- **Internationalization (i18n)** — Chinese/English bilingual support, reactive language switching
- **Task ID** — Auto-generated TAM-{N} format identifiers (com.tamarix.ticket_id)
- **Workflow Validation** — Task status transition legality check (front-end fallback with defined transition rules)
- **Full-text Search** — Task search page based on Matrix Room Message search
- **Assignee** — AssigneeSelect component, pick assignee from Room member list
- **Task Relations** — Support relation types: blocks / blocked-by / related / duplicates / subtask-of
- **Change History** — TaskHistory component, timeline display based on State Event changes
- **File Attachments** — FileUploadZone / AttachmentList / AttachmentPreview, upload/preview/delete support
- **Search Page** — Dedicated search route at /search

## Known Limitations

- Kanban intra-column drag sorting (card reordering within a column) not yet implemented
- No Application Service backend; ID generation and workflow validation are front-end fallback implementations
- End-to-end encryption (E2EE) not yet enabled
- Federated communication supported in theory but not tested in practice
