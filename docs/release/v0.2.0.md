# Tamarix v0.2.0

## Overview

Tamarix v0.2.0 completes the P2 Intelligence phase, adding notification system, worklog/time tracking, version management, rich text descriptions, member management, bulk operations, mobile adaptation, dashboard enhancements, and end-to-end encryption (E2EE) infrastructure.

Core concept: **One task = One Matrix Room**, providing built-in real-time collaboration, federated communication, and end-to-end encryption.

---

## Features

### P2 ‚Äî Intelligence (Continued)

#### Notification System

- **Notification Types** ‚Äî Assign, status change, due reminder, mention
- **Notification Parsing** ‚Äî `parseNotificationFromEvent()` extracts notifications from Matrix state events
- **Due Reminder** ‚Äî `isDueSoon()` checks tasks approaching deadline (within 24h) or recently overdue (within 7 days)
- **Notification Store** ‚Äî Reactive notification list with unread count, debounce deduplication, localStorage persistence for reminded IDs
- **Notification Panel UI** ‚Äî Bell icon + unread badge in AppHeader, Popover notification list, mark-as-read actions

#### Worklog / Time Tracking

- **WorklogEntry Type** ‚Äî userId, hours, note, loggedAt fields
- **State Event Persistence** ‚Äî `com.tamarix.worklog` custom state event for worklog entries
- **Worklog Store** ‚Äî Load/add/remove worklog entries, estimate-vs-actual comparison
- **WorklogPanel Component** ‚Äî Work log list + registration form + estimate vs actual summary in task detail page

#### Version Management

- **VersionInfo Type** ‚Äî name, description, startDate, releaseDate, released flag
- **State Event Persistence** ‚Äî `com.tamarix.version` and `com.tamarix.task_version` custom state events
- **Versions Store** ‚Äî CRUD for project versions with sync listener
- **Version UI** ‚Äî Version management in project settings page

#### Rich Text Description

- **Markdown Editor** ‚Äî Textarea + live preview + toolbar, powered by `marked` library
- **Description State Event** ‚Äî `com.tamarix.description` stores Markdown-formatted task description (separate from `m.room.topic`)
- **Task Interface Extension** ‚Äî `formattedDescription` field on Task

#### Member Management

- **MemberManager Component** ‚Äî View/remove project members, invite new members via Matrix user ID
- **Power Level Display** ‚Äî Shows member role (Admin / Moderator / Member)

#### Bulk Operations

- **BulkActionBar Component** ‚Äî Multi-select tasks, batch change status/priority/archive/add-tag
- **Kanban Multi-select** ‚Äî Shift+click to select multiple cards, checkbox + ring highlight for selected state
- **Task List Multi-select** ‚Äî Checkbox column in data table, bulk action bar on selection

#### Mobile Adaptation

- **Responsive Sidebar** ‚Äî Sheet drawer-style sidebar on mobile (shadcn Sidebar built-in support)
- **Kanban Mobile Layout** ‚Äî Vertical stacking with collapsible Accordion columns below `md:` breakpoint
- **Data Table Mobile Layout** ‚Äî Card list view on mobile, filter/sort as bottom Sheet
- **Task Detail Mobile Layout** ‚Äî Full-screen detail panel, bottom fixed tab bar, collapsible metadata Accordion
- **Touch Optimization** ‚Äî Long-press (300ms) to trigger drag, 44px minimum touch targets, drag handle icons on mobile
- **use-long-press Utility** ‚Äî `use-long-press.svelte.ts` composable for long-press gesture

#### Dashboard Enhancement

- **ProjectProgressCard** ‚Äî Per-project progress statistics (todo/in_progress/review/done counts)
- **TeamWorkloadCard** ‚Äî Member workload distribution chart
- **OverdueTaskList** ‚Äî List of overdue tasks with due date display

#### Notification Preferences

- **NotificationPrefs Type** ‚Äî Per-room notification preferences: assign, status change, due remind, mention; channels (in_app / email)
- **State Event Persistence** ‚Äî `com.tamarix.notification_prefs` custom state event
- **Preference UI** ‚Äî Notification settings in project settings page

#### Task Watching

- **Watcher State Event** ‚Äî `com.tamarix.watcher` for explicit task follow/subscribe
- **getWatchers()** ‚Äî Read watcher list from room state

#### Recent Tasks

- **Recent Tasks Store** ‚Äî Track recently visited tasks, localStorage persistence (max 20 entries)
- **RecentTaskEntry** ‚Äî taskId, taskTitle, projectRoomId, visitedAt

#### Attachment Deletion

- **Delete Button** ‚Äî Red delete button on each attachment item
- **Confirmation Dialog** ‚Äî AlertDialog before redacting the Matrix event
- **i18n** ‚Äî Delete confirmation/success/failure messages

### P2 ‚Äî E2EE Infrastructure

- **Task Encryption** ‚Äî Optional encryption when creating tasks (`encrypted` flag + `m.room.encryption` initial state with `m.megolm.v1.aes-sha2`)
- **Project Encryption Propagation** ‚Äî Encrypted projects create encrypted template child rooms
- **Encryption Detection** ‚Äî `isRoomEncrypted()` checks `m.room.encryption` state event, auto-populated in `roomToTask()`
- **UI Indicators** ‚Äî Lock icon on encrypted Kanban cards, encrypted badge on task detail page
- **Create Dialog Toggle** ‚Äî Switch + Lock icon in TaskCreateDialog and project creation form

> **Note:** Matrix Spaces (projects) do not support E2EE; only child rooms (tasks) can be encrypted. E2EE protects message flow (comments, attachments) only; state events (`com.tamarix.*`) remain plaintext. Encrypted rooms cannot be read by AS Bot ‚Äî P3 AS enhancement must account for this limitation.

## Known Limitations

- Kanban intra-column drag sorting (card reordering within a column) not yet implemented
- No Application Service backend; ID generation and workflow validation are front-end fallback implementations
- E2EE protects message flow only; state events (`com.tamarix.*`) remain plaintext
- Federated communication supported in theory but not tested in practice
- Email notification channel defined in types but not implemented (front-end only supports in-app)
